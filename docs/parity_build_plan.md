# Parity Build Plan — remaining IO metrics

Per-metric checklist for bringing the remaining metrics to legacy parity and wiring them into the `[IO] Performance Metrics Pipeline` Databricks job. Generated from the `metrics-build-readiness` review fan-out (one assessor + one adversarial verifier per metric). Companion to `parity.md` (which tracks *validated* divergences) and the migration rule (reproduce legacy byte-for-byte before the `2026-07-01` cutover).

**Adherence is the reference implementation** (`metrics_data/adherent_time.py`): phantom-adherence cutover, meeting/leave `dimensioned_activity` filter, DIME-squad filter, quality/planning inclusion.

## Cross-cutting themes

- **Tier 1 — won't run:** `ntpj`, `tnps`, `content_csat` are pandas but the pipeline is PySpark; plus build-script bugs (`append_missing_dime_slots` arity, missing `read_adjustment_csv`).
- **Tier 2 — slot filters:** `normalized_occupancy` + `shrinkage` need the meeting/leave and/or DIME-squad filters (occupancy's DIME-squad list also includes `social`).
- **Tier 3 — era-gating & granularity:** legacy date floors (`>=2025-12-01`, `>=2026-03-01`), 2025→Dec-2025 roster pinning, and week+month-only materialization for the composites/`wows`/`tnps`.
- **Tier 4 — metric-specific legacy quirks** (per metric below).

Build **one metric at a time**: apply fixes → wire into the job → run → parity-check → tick it off.

> Legend: each metric lists its **final verdict**, the **adversarially-confirmed fixes**, and **other parity risks** to verify once it runs.

## `normalized_occupancy`  —  **NEEDS-FIXES**

_normalized_occupancy = per-agent occupancy (SUM occupancy_minutes / SUM required_minutes) divided by a (month, district, shift) benchmark that is the equal-weight mean of the per-squad occupancy ratios — this matches docs/metrics_definitions.md (#3) and the legacy two-step AVG(occupancy_monthly) benchmark. HOWEVER the build is NOT parity-ready because the legacy DIME-squad exclusion is entirely MISSING from both layers, which inflates the denominator and breaks byte-for-byte parity._
